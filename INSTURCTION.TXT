INNERTUBE API — COMPLETE IMPLEMENTATION

Base URL for all requests: https://www.youtube.com/youtubei/v1/

Use this exact client context object in the body of every single Innertube request:

context: {
  client: {
    clientName: "WEB",
    clientVersion: "2.20240101.00.00",
    hl: "en",
    gl: "IN",
    userAgent: "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36",
    clientFormFactor: "UNKNOWN_FORM_FACTOR"
  }
}

Implement all of these API functions in lib/innertube.ts:

searchVideos(query, continuationToken) — POST to /youtubei/v1/search with body containing the context and query field. Parse sectionListRenderer > itemSectionRenderer > contents to extract videoRenderer and channelRenderer items. Support pagination via continuationToken.

getVideoData(videoId) — POST to /youtubei/v1/player with body containing context, videoId, racyCheckOk: true, contentCheckOk: true. Returns videoDetails object with title, author, lengthSeconds, viewCount, shortDescription, thumbnail, isLive, isLiveDvr fields. Also returns streamingData with formats array (muxed streams) and adaptiveFormats array (separate video-only and audio-only streams). Parse all itag numbers from adaptiveFormats. Also try using ANDROID client (clientName: "ANDROID", clientVersion: "20.10.38") as fallback if WEB client streams are cipher-protected.

getNextData(videoId) — POST to /youtubei/v1/next with context and videoId. Returns related videos in secondaryResults and comments in engagementPanels. Parse both.

getHomeFeed(continuationToken) — POST to /youtubei/v1/browse with body containing context and browseId: "FEwhat_to_watch". Parse richGridRenderer > contents to get richItemRenderer > content > videoRenderer items.

getTrending(category) — POST to /youtubei/v1/browse with browseId: "FEtrending". Support tabs for Now, Music, Gaming, Movies, News by passing the correct params field from the tab data.

getChannelPage(channelId) — POST to /youtubei/v1/browse with browseId set to the channelId string. Parse header for channel name, subscriber count, banner, avatar. Parse tabs for Videos, Playlists, Community, About sections.

getPlaylist(playlistId) — POST to /youtubei/v1/browse with browseId: "VL" + playlistId. Parse playlistVideoListRenderer for the video list.

getTranscript(videoId) — POST to /youtubei/v1/get_transcript with context and params field constructed from videoId. Parse transcriptSegmentListRenderer for timed text segments.

getSearchSuggestions(query) — GET to https://suggestqueries-clients6.youtube.com/complete/search?client=youtube&q={query}&callback=func. Parse the JSONP response to extract suggestion strings.

getComments(videoId, continuationToken) — POST to /youtubei/v1/next with the correct continuation token extracted from initial video data. Parse commentThreadRenderer items for comment text, author, likeCount, replyCount.

getLiveChatReplay(videoId) — POST to /youtubei/v1/live_chat/get_live_chat_replay with context and videoId for livestream chat.

STREAM HANDLING AND QUALITY SELECTION

YouTube's adaptiveFormats array contains separate video-only streams and audio-only streams identified by their itag number. You must implement MSE (MediaSource Extensions) to combine them for playback.

The important itag numbers to support are:

For video streams: itag 137 is 1080p MP4, itag 248 is 1080p WebM, itag 136 is 720p MP4, itag 247 is 720p WebM, itag 135 is 480p MP4, itag 244 is 480p WebM, itag 134 is 360p MP4, itag 243 is 360p WebM, itag 160 is 144p MP4.

For audio streams: itag 140 is 128kbps AAC MP4 (Medium quality), itag 251 is 160kbps OPUS WebM (High quality — best for music), itag 139 is 48kbps AAC MP4 (Low quality).

For muxed streams (video+audio combined, simpler to implement as fallback): itag 22 is 720p MP4, itag 18 is 360p MP4. Use these as fallback if MSE fails.

Stream URL deciphering: Some stream URLs in adaptiveFormats are direct URLs (signatureCipher is absent). If signatureCipher is present, you must decipher it. To decipher: fetch the YouTube player JS file URL from the initial page response, extract the signature function name and helper object from the JS, run the decipher operations on the cipher string to produce the valid URL. Implement this in lib/decipher.ts. Cache the player JS per session to avoid re-fetching.

Implement the QualitySelector component as a dropdown that shows all available video quality options and separately all available audio quality options. When user changes quality, switch the MSE source buffers without interrupting playback — seek to currentTime after switching.

INNERTUBE API — COMPLETE IMPLEMENTATION
All Innertube calls must be server-side in Next.js Route Handlers. Never call from the browser. Base URL is https://www.youtube.com/youtubei/v1/.
Client context to use in every request body:
context.client.clientName = "WEB"
context.client.clientVersion = "2.20240101.00.00"
context.client.hl = "en"
context.client.gl = "IN"
context.client.userAgent = "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/124.0.0.0 Safari/537.36"
If WEB client returns cipher-protected stream URLs, retry the same request using ANDROID client with clientName "ANDROID" and clientVersion "20.10.38" — ANDROID client returns direct unciphered stream URLs in most cases.
Implement all these functions in lib/innertube.ts:
searchVideos(query, continuationToken) — POST /youtubei/v1/search, body has context and query. Parse sectionListRenderer deeply to extract all videoRenderer and channelRenderer items. Return array of {videoId, title, thumbnails, duration, viewCount, channelName, channelId, channelAvatar, publishedTime, description}.
getVideoPlayer(videoId) — POST /youtubei/v1/player, body has context, videoId, racyCheckOk true, contentCheckOk true. Extract videoDetails and streamingData. From streamingData.adaptiveFormats, group streams by whether mimeType includes "video" or "audio". Extract itag, mimeType, quality, bitrate, url, audioQuality, width, height for each stream. Also extract streamingData.formats (muxed) as fallback. Return all of this structured data.
getNextData(videoId) — POST /youtubei/v1/next, body has context and videoId. Parse secondaryResults for related videos. Parse engagementPanels for chapters (look for engagementPanelSectionListRenderer with panelIdentifier containing "chapters"). Parse chapters from macroMarkersListItemRenderer items — each has title, timeRangeStartMillis. Parse comments from commentSectionRenderer but main comments load via continuation. Return {relatedVideos, chapters, commentsContinuationToken}.
getHomeFeed(continuationToken) — POST /youtubei/v1/browse, browseId "FEwhat_to_watch". Parse richGridRenderer contents for richItemRenderer items containing videoRenderer.
getTrending() — POST /youtubei/v1/browse, browseId "FEtrending". Parse expandedShelfContentsRenderer contents.
getChannelPage(channelId) — POST /youtubei/v1/browse, browseId is the channelId string. Extract channelMetadataRenderer for name, description, avatar. Extract c4TabbedHeaderRenderer for subscriber count, banner. Parse tabs array for Videos tab content.
getPlaylist(playlistId) — POST /youtubei/v1/browse, browseId "VL" + playlistId. Parse playlistVideoListRenderer for video list with playlistIndex included.
getComments(continuationToken) — POST /youtubei/v1/next, body has context and continuation token in the continuation field. Parse commentThreadRenderer items. Each has comment.runs for text, authorText for name, authorThumbnail for avatar, voteCount for likes, replyCount.
getTranscript(videoId) — POST /youtubei/v1/get_transcript, body has context and params field. Params is constructed by base64 encoding a protobuf with the videoId. Parse transcriptSegmentRenderer items for startMs and snippet text.
getSearchSuggestions(query) — GET to https://suggestqueries-clients6.youtube.com/complete/search with params client=youtube, q=query, callback=? and parse the JSONP array.
